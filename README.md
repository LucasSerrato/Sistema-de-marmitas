# 🍱 Healthy Meal Ordering System

## 📘 Academic Project Overview

This system was developed as part of the **Integrative Project: Object-Oriented Systems Development** course at **SENAC**. The goal was to design and implement a web-based platform that promotes **healthy eating habits** by providing users with **nutritional guidance** through a **BMI (Body Mass Index) calculator** and **fitness meal suggestions**, tailored to individual health profiles.

The project aligns with academic objectives by combining **technical skills**, **problem-solving**, and **real-world application development**. It demonstrates an understanding of the **Object-Oriented Programming (OOP)** paradigm, **modular design**, and **client-server architecture**, reflecting best practices in software engineering.

Furthermore, the system features a **fully functional CRUD implementation** with **validated login access**, ensuring that users are authenticated before accessing the platform. The **front-end and back-end are fully integrated**, allowing seamless interaction between the interface and server-side logic.

---

## 🎯 Project Objectives

- Promote healthy lifestyles and nutritional awareness through technology.
- Apply object-oriented programming concepts in a real-world system.
- Develop a responsive and user-friendly web interface.
- Implement data processing based on user inputs (weight, height, gender).
- Suggest fitness meals according to BMI results.
- Implement secure login validation and CRUD operations.
- Ensure seamless front-end and back-end integration.

## 💡 Key Concepts Applied

- **Object-Oriented Programming (OOP)**: Use of encapsulation, abstraction, inheritance, and polymorphism to structure classes and data models.
- **Client-Server Architecture**: Separation of concerns between the user interface (client) and application logic (server).
- **Full CRUD Implementation**: Create, Read, Update, Delete operations functional across the application.
- **Login Validation**: Form inputs are verified to ensure secure authentication.
- **Academic Research**: Nutritional content and recommendations were developed using guidance from verified sources and national health references.

---

## 🧩 System Features

- ✅ **BMI Calculator** – Calculates BMI based on user inputs.
- ✅ **Nutritional Classification** – Categorizes BMI into ranges (underweight, normal, overweight, obesity).
- ✅ **Fitness Meal Recommendations** – Offers healthy meals aligned with BMI results.
- ✅ **Home Screen** – Displays health tips and platform instructions.
- ✅ **Login and Access Control** – Validates login credentials before granting access.
- ✅ **Feedback Area** – Users can submit comments, opinions, or suggestions.
- ✅ **BMI History Tracking** – View past BMI calculations (optional for future expansion).
- ✅ **CRUD Functionalities** – Users can create, edit, view, and delete records with proper form validation.

# Sistema de Marmitas Saudáveis

## 📘 Visão Geral do Projeto Acadêmico

Este sistema foi desenvolvido como parte do curso de **Projeto Integrador: Desenvolvimento de Sistemas Orientado a Objetos** no **SENAC**. O objetivo principal foi criar uma plataforma web capaz de **promover hábitos alimentares saudáveis**, fornecendo **orientação nutricional** por meio de uma **calculadora de IMC (Índice de Massa Corporal)** e sugestões de **marmitas fitness** personalizadas conforme o perfil nutricional do usuário.

O projeto atende aos objetivos acadêmicos ao combinar **habilidades técnicas**, **resolução de problemas** e desenvolvimento de aplicações com propósito social. Demonstra domínio da **Programação Orientada a Objetos (POO)**, **design modular** e uso da **arquitetura cliente-servidor**, em conformidade com as boas práticas da engenharia de software.

Além disso, o sistema apresenta uma implementação completa de **CRUD funcional**, com **validação de login**, garantindo que apenas usuários autenticados possam acessar a plataforma. O **front-end está totalmente conectado ao back-end**, permitindo uma interação fluida entre a interface e a lógica de servidor.

---

## 🎯 Objetivos do Projeto

- Promover estilos de vida saudáveis e educação nutricional por meio da tecnologia.
- Aplicar conceitos de POO em um sistema prático.
- Criar uma interface web amigável e responsiva.
- Processar dados dos usuários (peso, altura, sexo) e gerar resultados personalizados.
- Implementar autenticação de login com validação.
- Desenvolver operações completas de CRUD.
- Integrar eficientemente o front-end e o back-end.

---

## 💡 Conceitos Aplicados

- **POO (Programação Orientada a Objetos)**: Uso de encapsulamento, abstração, herança e polimorfismo.
- **Arquitetura Cliente-Servidor**: Separação entre interface gráfica (cliente) e lógica de negócio (servidor).
- **CRUD Completo**: Operações de Criar, Ler, Atualizar e Excluir implementadas com sucesso.
- **Validação de Login**: Credenciais são verificadas antes do acesso ao sistema.
- **Pesquisa Acadêmica**: Regras e sugestões nutricionais baseadas em fontes confiáveis.

---

## 🧩 Funcionalidades do Sistema

- ✅ **Calculadora de IMC** – Cálculo com base em peso, altura e sexo.
- ✅ **Classificação Nutricional** – Interpretação dos resultados (magreza, normal, sobrepeso, obesidade).
- ✅ **Sugestão de Marmitas** – Sugestões baseadas nos dados do IMC.
- ✅ **Tela Inicial** – Informações sobre o sistema e boas práticas.
- ✅ **Login com Validação** – Autenticação segura para acesso.
- ✅ **Área de Feedback** – Envio de sugestões e comentários.
- ✅ **Histórico de IMC** – Exibição de consultas anteriores (em expansão).
- ✅ **CRUD Funcional** – Manipulação de dados com validações em formulários.
